电动机原理

所谓电动机，就是用电产生动力的机，其原理就是电磁力之间的转换,通过电磁⼒转换实现运动。

无刷电机的核心结构

一个简单的无刷电机模型，主要有两部分构成，一个是 定子 有三个线圈和一个永磁铁线圈固定不动，还有一个就是 转子 ，就是外围的永磁铁。本章主要讲的是正弦波驱动

永磁同步PMSM

PMSM特性 正弦波
反电动势 具有正弦波反电动势
运行电流 正弦波电流
绕组方式 定子绕组为分布绕组，永磁转子形成正弦磁场
控制方式 控制算法复杂(FOC)
运行方式 绕组三相全部导通

驱动方法 

1

通过改变线圈通电⽅向切换磁极，推动永磁铁旋转

线圈固定不动， 永磁铁可以转动。我们知道线圈都可以当做电磁铁。只要给线圈通电，它就会变得和磁铁一样。磁铁同极相斥，异极相吸，所以永磁铁会被电磁铁朝外的两极吸引和排斥。最终达到平衡位置。如果把电磁铁的电源反过来接，它的磁极也会反过来。然后永磁铁也会转到相反的位置，好复习完这些知识，我们就能明白，只用三个线圈，怎么让永磁铁转起来了？

六步换相驱动法

通过依次变换 A点 、B点、C点 三点的电势差，从 $U_{AB}$ -> $U_{CB}$ -> $U_{CA}$ -> $U_{BA}$ -> $U_{BC}$ -> $U_{AC}$按照这个顺序通电。只需要六步，就能让磁铁转一圈，一直重复这一过程，它就转起来了。是不是很简单？我们把电机能转的部分叫做转子固定的部分，叫做定子。把切换三根相线电压正负的过程称之为换相。 电机转一圈需要换向六次，这就是六步换向驱动法了。

GPIO控制三相半桥电路实现无刷电机换相

三极管三个极 

• B = Base  →  基极（控制极）

• E = Emitter  →  发射极（载流子发射端）

• C = Collector  →  集电极（载流子收集端)

简单讲，B 控制，E 流出，C 流入（NPN），B 控制，E 流入，C 流出（PNP）

MCU的逻辑信号分别接入PNP型三极管、NPN型三极管的控制极（Base基极），上面的PNP型三极管的E发射极接外部上拉电源3.3V，为V_OUT 提供参考高电压 而下面的NPN型三极管的E发射极接 GND, 为V_OUT 提供参考低电压，当二者都不导通时，V_OUT处于悬空状态

GPIO_1 GPIO_2 PNP NPN V_OUT
0 0 导通 截止 接入3.3V
0 1 导通 导通 同时接入3.3V & GND，电源直接短接，三极管爆炸
1 0 截止 截止 悬空状态
1 1 截止 导通 接入GND

⚠️需要注意：当上管和下管直接导通时，电流远远超过500毫安，远超芯片手册的额定功耗P=V×I。所以只需要几秒钟的时间，三极管就烧糊，所以我们在写软件逻辑时，必须学会谨慎操作。因为一个小失误，真的有可能直接烧坏元件。上电之前先排除掉可能出现的意外，多用万用表测测。

三相半桥电路 

六步换相控制：通过GPIO控制三相半桥电路实现⽆刷电机换相。
相序影响：三相接线顺序仅影响电机正反转⽅向，可任意连接。
⽅向调整：调换任意两相接线可反转电机旋转⽅向。
控制端连接：上管控制端接`GPIO_PA8-10`，下管控制端接`GPIO_PB13-15`。
PWM输出口：六个IO口均为⾼级定时器TIM1的PWM输出通道，为后续拓展预留。
供电要求：需连接板载5V系统供电。

void Delay_ms(uint16_t ms)
{
    uint32_t i, j;
    for(i = 0; i < ms; i++)
    {
        for(j = 0; j < 8000; j++); 
    }
}

{

  RCC_APB2PeriphClockCmd(RCC_APB2Periph_TIM1, ENABLE);
  RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA | RCC_APB2Periph_GPIOB, ENABLE);


  GPIO_InitTypeDef GPIO_InitStructure;
  GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_PP;

  GPIO_InitStructure.GPIO_Pin = GPIO_Pin_13 | GPIO_Pin_14 | GPIO_Pin_15;

  GPIO_ResetBits(GPIOB, GPIO_Pin_13 | GPIO_Pin_14 | GPIO_Pin_15);

  {
   Delay_ms(time_set);

     C_L_SET;
    break;
    case 5:
     A_H_SET;
     B_0_SET;
    break;
    }
  }
}

反电动势原理

首先电机的原理已经有所了解，就是电转势、势转力的过程，而反过来，力也可转势、再势转电，所以无刷电机也可以是发电机。

无刷电机的三条逻辑线任意选两根线接到 LED 上，用手拨动电机，会发现LED神奇的亮起来了，而且随我们拨动越快，LED就越亮， 通过示波器的探针接到这两根线，我们可以看到，转电机的时候会产生正弦变化的电压，这就是 反电动势。

从示波器视图中可以发现一些规律，比如转得越快，电压越高，频率越快

○ 电机兼具电动和发电特性，转动无刷电机可产生正弦变化的反电动势，转速越快，反电动势电压越高、频率越高，其幅值与转子转速正相关。

电磁感应定律

$U_{b}(θ) = n B S w cos θ$

$U_{b}$ 表示电机反电动势 θ表示转子磁场和线圈横截面的角度 n 表示线圈匝数 B表示匀强磁场强度 S表示线圈横截面积 w表示转子转速

其中n 、B、S 都是电机自身固定的参数，我们只需要看 欧米伽w 、 西塔θ ，这二位美女就可以了。

转速欧米伽越大反电动势幅值越大，同时角度西塔变化也越快，波形也越密集，这与之前实验观察的结论一致。现在如果我们在外部接入一个电源，使其产生正向电流，让它变回一个电动机。

设定$U_{A}$是电源，电压$U_{B}$是反电动势，$U_{r}$是线圈内阻分压。根据基尔霍夫电压定律，在同一回路中，两个电压互为相反数可得：$U_{A}+U_{B}+U_{r}=0$

基尔霍夫电压定律

由基尔霍夫电压定律推导电路电压关系，明确电源电压、反电动势、线圈内阻分压的关联，且需先界定回路和方向再分析电压。

在下图中，我们设定顺时针为正方向电源，在这个回路中的电压等于下侧节点电势。减去上侧节点电势，所以是负数，而电流与参考方向相同，是正数。电流与电压相乘，是负数。所以电能做负功。在电源内部表现为电池化学能转化为电能。

这才与常理一致。明白这个原理，就知道这个等式为什么这么写了，我们回到熟悉的六步换向对三个线圈分别列出电磁感应公式。设定A线圈与磁场角度为西塔θ，转子转速为欧米伽w，正极朝外,就有 $U_{Ab}(θ) = n B S w cos θ_{A}$ ，因为共用同一转子，欧米伽w相同,而 西塔θ 是三个线圈与同一转子的角度，三个线圈两两互为120°呈现旋转对称，按照同样设定列出其他两个公式，都是正极朝外，所以有$θ_{A} = θ_{B} - 120° = θ_{C} + 120° $

只要导体（绕组线圈）在磁场中做相对运动，切割磁感线，就会在导体两端感应出电动势（电压），和导体里有没有电流、有没有接负载，完全没有关系。

就像发电机：

●	发电机的线圈在磁场里转动，哪怕你不接任何负载、线圈里没有电流，线圈两端依然会感应出电压，这就是反电动势的本质。

●	电机的绕组，本质就是发电机的线圈：只要转子带着磁场在转动，绕组就会切割磁感线，持续产生反电动势，和绕组有没有通电、有没有电流，完全是两个独立的物理过程。

三相

图里的状态是：A、B相导通（有电流，驱动电机），C相悬空（无电流） 转子在持续转动：A、B相的电流产生的磁场，推着转子磁场旋转，整个电机在高速转动。 C相绕组一直在切割磁感线：C相的线圈，和转子的永磁磁场一直在做相对运动，持续切割磁感线。 电磁感应产生$ U_{Cb}$：根据法拉第定律，C相绕组两端会感应出反电动势$ U_{Cb}$，哪怕C相悬空、线圈里没有任何电流，这个电压依然存在。 为什么要悬空？ 只有让C相完全悬空、没有电流，C相的端电压才只由反电动势决定（没有电阻压降 ），我们才能通过ADC采样C相电压，精准检测反电动势的过零点，判断转子位置。

三个定子线圈呈120°旋转对称，共用同一转子，反电动势满足旋转对称的三角函数规律，可通过公式2UC-UB-UA=3UCB消除内阻分压，得到纯粹的反电动势。

力矩与六步换向的角度优化

线圈在磁场中的力矩随与转子磁场的角度变化，0°时力矩最大，90°时为0，180°时反向最大；无刷电机始终两个线圈同时通电，合力矩等效于一个根号三倍的虚拟线圈。

六步换向每步转子转过60°，为提升力矩效率，需将最大扭矩点落在60°区间中点，即换向初始角度-30°、结束角度+30°。

虚拟线圈的轴向角度与悬空线圈的径向角度一致，悬空线圈反电动势过零点出现在轴向角度-90°，且检测到过零点后，需等转子再转30°再换向。

反电动势波形的实验验证

强制六步换向实验：打印出的反电动势波形能观察到转子振荡、自感脉冲，但因虚拟线圈角度为-60°到0°，错过过零点，无法检测。

三步换向实验：让线圈磁场一次转120°（步序加二），转子追赶阶段为-120°到0°，成功捕捉到悬空线圈-90°处的过零点，且波形峰值更大，验证了过零点的存在。

实验中观察到换向时线圈断开会产生高压尖峰，但通过50%PWM长时间测试，IO口可承受该瞬时电流和电压。

过零检测与电机启动/闭环控制方案 

○	启动方案：先强制换向到第零步并保持100毫秒，让电机归位且振荡消失；再换向到第二步保持5毫秒，转子转过30°经过首个不明显过零点；最后换向到第三步，线圈领先转子150°，转子持续加速并出现清晰过零点，此时接入闭环控制。

○	闭环过零检测换向：检测到反电动势过零点后计时，计时达到换相间隔一半时换向，循环“等待过零点-等待半周期延时”两个环节；需先设定换相周期初始值，再通过统计两次换向间隔时长持续刷新周期。

○	干扰解决：过零检测中易受线圈自感尖峰、电磁干扰影响出现丢步和振荡，可采用软件滤波（连续四次判定过零为真再动作）过滤脉冲干扰。

根据反电动势波形精确控制六步换相

产生 通过检测反电动势波形，实现过零检测算法

在找零点之前，我们首先了解力矩的概念，回到上面六步换相，对 A线圈 的电磁感应公式，将 A设定为 H状态， 此时设定 A线圈 与转子的磁场角度为西塔θ， 转子转速为欧米伽ω,根据右手定则得出， A线圈 正极朝外，就有$U_{Ab}=nBSωcosθ_{A}$ 同样设定下，分别让 B、C 设定为 H状态,得出都是正极朝外的结果。 其中，转速欧米伽，由于三个线圈共用一个转子，所以三者的欧米伽ω相同。 而 线圈与转子的磁场角度西塔θ是三个线圈与同一个转子的角度，三个线圈两两互为120°呈现旋转对称，所以有下面规律：

$ θ_{A}$ = $θ_{B}$ - 120° $θ_{A}$ = $θ_{C}$ + 120°

力矩的概念

安培定律

安培定律推到出线圈在匀强磁场的力矩公式

力 公式 Remark
力矩M M=nBSIcosθ 力矩公式中包含电流 I,θ 为法线与磁场夹角
感应电动势E U=nBSωcosθ θ 为定子横截面与转子磁场夹角

结果匀强磁场的力矩公式和电磁感应公式展示出惊人的一致性，这并非巧合。

爱因斯坦在狭义相对论已经论证了，磁场和电场是不同参考系下的同一个实体，电磁感应定律和洛伦公式描述的其实是同一个物理现象。 根据牛顿第三定律得出，线圈受到的力矩和转子的受到的力矩等向反大 (去除非匀强磁场的影响)

θ 力矩大小
0° 正向最大
90°零点位置 减至零
180° 反向最大

而电机要的就是力矩，只有 0° 时，力矩最大，效率最高,这是无刷电机工作的关键

虚拟线圈的最大扭矩角度

无刷电机工作的原理：两个线圈同时通电，夹角为120度，最后一个线圈处于悬空状态

和前面论证单个力矩不同的是，无刷电机总有两个线圈同时通电，所以我们先要计算两个力矩的合力 首先是两个线圈西塔θ夹角为120°，A线圈和B线圈的电流方向互为相反数

力矩 公式
$M_{A}$ $nBSIcosθ_{A}$
$M_{B}$ $-nBSIcosθ_{B}$
$M_{N}$= $M_{A}$ + $M_{B}$ $ \sqrt{3} $ nBSI($cosθ_{B}$ - 30°)

上面可得，两个线圈的合力等于一个$ \sqrt{3} $ 的线圈摆在下图这个位置，结果和之前的磁铁定性分析一致。

虚拟线圈的最大扭矩角度

六步换相角度

每一步转子转过60°，最大效率就是让扭转点放在60°区间的中点，此时力矩最大

换相优化

也就是说，按六步换相来讲，每一步刚开始时的角度为-30°，结束角度为+30°时，换相角度获得力矩为最高效率

虚拟线圈

为了直观，我们用虚拟线圈的轴向直线和转子的角度来分析。那么，前面的径向角度-30°到+30°度就变成了轴向的-120°到-60°

除了直观还有一个好处，就是虚拟线圈的轴向角度恰好等于悬空线圈的径向角度。所以B线圈的径向角度也是-120°到-60°。而这个区间对应的反电动势曲线如图。其中点-90°，就是过零点。

好消息是$U_{B}$确实经过了零点。坏消息是这个零点没那么贴心，并不是说检测到零点就可以换相了。而是要在零点出现之后，再等转子转过30度才开始换向。我们把虚线平移到这里，当然角度还是不变的。我们只要知道两个通电线圈的连线与转子磁场的角度，就是悬空线圈反电动势的角度就行了

过零点分析

  • 过零点分析：通过虚拟线圈轴向直线和转⼦⻆度进⾏直观分析。
  • ⻆度转换：径向⻆度负30度到正30度对应轴向负120度到负60度。
  • 虚拟线圈特性：虚拟线圈轴向⻆度等于悬空线圈径向⻆度。
  • 反电动势曲线：B线圈径向⻆度负120度到负60度对应特定反电动势曲线。
  • 过零点位置：反电动势曲线中负90度为过零点。
  • 换相时机：检测到过零点后需等待转⼦再转过30度才开始换向。
  • ⻆度关系：通电线圈连线与转⼦磁场⻆度等于悬空线圈反电动势⻆度。

反电动势的求出

我们需要想办法使用三个ADC检测的电压将反电动势算出来，根据基尔霍夫定压定律有: $(U_{B}-U_{C}) + U_{Cb}-U_{Bb} + U_{Br} =0$。

又因为三个线圈的反电动势是旋转对称，他们的正极都是朝外，通过消除内阻分压，利用三角函数诱导公式简化得到纯粹的反电动势

过零检测所需的所有信息

反电动势的波形

用串口助手将反电动势波形打印出来

从 转子震荡和自感脉冲曲线，看三个相悬空时的反电动势波形，从每个相悬空时的波形可以看出，找不到发生在-90度时的零点

由于六步这种强制换相，每次换相时，虚拟线圈的角度在-60°到0°，早就错过-90°过零点

震荡曲线的构成

换相刚开始，由于通电线圈突然断开，产生高压尖峰，电流流经三极管基极Base和IO 的保护二极管释放掉，直接让IO口承担高达600mA的瞬时电流，5.5V的瞬时高压

分析反电动势的变化与电机的关系

换相刚开始，线圈超前转子60度 转子受到扭矩开始追赶、加速，转子在赶上线圈之前，扭矩不断减小（即加速度不断减小的加速运动） 追赶上之后，轴向角度为0度 由于惯性，转子继续向前转，开始做减速度逐渐增大的减速运动，直到速度减到0 开始反向加速，如此循环

反电动势的变化

在追赶阶段，悬空线圈的径向角度是-60°到0° 按曲线来看，反电动势的初值是正数且逐渐增大， 但刚起步的时候，进行的是初速度为0的加速度运动，直到转子追赶上线圈（此时吸力最大），电压达到峰值，速度达到最大，之后转子超过线圈，开始减速，次数的余弦曲线开始不断减小，转速也不断减小，根据电磁感应定律，$U = n B S w cos θ$，转子由于惯性超过0度，继续向前转，开始做减速度逐渐增大的减速运动，直到速度减到0，开始反向加速，如此循环，欧米伽不断减小，到0，再不断增大，每次势能不断减小，直到为0，电压直线下滑，0，反向增大，以此类推，电压过程呈现周期性重复变化

零点在哪

前面我们已经知道，-90度的时候就是悬空线圈的过零点，所以使用六步换相是行不通的

我们使用三步换相，每次让线圈磁场转120度，所以在程序上只要让步序一次加2就可以，从下面曲线得出，相比较之前，电压尖峰更大了，此时由于转子的加速行程翻倍，追赶阶段变成了从-120度到0度。

运动分析

前面我们已经知道，-90度的时候就是悬空线圈的过零点

转子从-120度开始，从速度0开始加速，而余弦曲线的绝对值却是逐渐减小的，所以反电动势的绝对值从速度0开始增加的，到转速增速比不过曲线（电磁感应）的降速，（电✌️势）反电动势开始减小，在-90度减到0度，之后余弦曲线随角度增加而增加，同时随着加速度正向增大，转速持续增大，电压持续增大，到0度最大，反复如此

程序设计

如果电机转速太慢，就检测不到清晰的反电动势了，而如果角度不对（-60度到0度），也检测不到过零点，好了，说了这么多，开始我们的设计之旅，如果设计出一个有效的电机启动方案呢。

串口打印时间间隔：0.1ms，从下图得出，电机从静止到零点用时大约用5ms时间，期间转了大概30度，此时反电动势曲线很清晰，这个时候我们就可以切换到闭环过零点换相（转子平稳旋转的前提下，检测到反电动势的零点开始计时，当计时等于换相时间间隔的一半的时候再次换相）

计时的作用

在转子过零点时，等待转子转过大约30度的时候在换相，换相之后的下一步时等待过零点，如此循环，整个过程分为两个环节，一个是等待过零点，一个是等待半周期延时

电机启动方案

首先先强制换相到第0步（A 0 ， B L ， C H），并持续100ms，目的让电机转到位并等待震荡消失, 而后直接换相到第二步（A H， B 0 ， C L），让线圈领先转子120度并保持5ms ，期间转子转过大约30度，经过第一个零点（-90度），此时过零点不明显，我们不用看，而是直接换相到第三步（A 0， B H， C L），此时线圈领先转子150度，转子转转过60度出现第二个零点，而60度区间里，转子将会持续加速，这样既可以保证不失步，又可以获得清晰的过零点，所以在第三步的时候，直接接到闭环控制的等待过零点环节就可以了。

接入闭环

我们需要给换相周期一个差不多的初始值，之后的闭环流程中，需要统计两次换相间隔时长，而后刷新这个周期，这样完整的闭环控制就完成了

线圈自感尖峰，堵转，电磁干扰，都会造成电机停转蜂鸣现象，而这里的高频均匀的震荡，很可能受到尖峰电压的干扰，此时我们做一个简单的软件滤波就够用了

定义一个计数，在过零检测判为true时不再立即动作，而是计数，连续判定四次为真的时候再动作，这样足够过滤绝大多数脉冲干扰了

void GPIO_Init_Config(void)
{
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA | RCC_APB2Periph_GPIOB, ENABLE);

    GPIO_InitTypeDef GPIO_InitStructure;

    // PA8 PA9 PA10 PNP上桥

    ALL_OFF(); 
}

void ADC_Init_Config(void)
{
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_ADC1, ENABLE);

    ADC_InitTypeDef ADC_InitStructure;
    GPIO_InitTypeDef GPIO_InitStructure;

    // 假设反电动势采样 = PA0
    GPIO_InitStructure.GPIO_Pin   = GPIO_Pin_0;
    GPIO_InitStructure.GPIO_Mode  = GPIO_Mode_AIN;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    ADC_InitStructure.ADC_Mode               = ADC_Mode_Independent;
    ADC_InitStructure.ADC_ScanConvMode       = DISABLE;
    ADC_InitStructure.ADC_ContinuousConvMode = DISABLE;
    ADC_InitStructure.ADC_ExternalTrigConv   = ADC_ExternalTrigConv_None;
    ADC_InitStructure.ADC_DataAlign          = ADC_DataAlign_Right;
    ADC_InitStructure.ADC_NbrOfChannel       = 1;
    ADC_Init(ADC1, &ADC_InitStructure);

    ADC_Cmd(ADC1, ENABLE);
    ADC_ResetCalibration(ADC1);
    while(ADC_GetResetCalibrationStatus(ADC1));
    ADC_StartCalibration(ADC1);
    while(ADC_GetCalibrationStatus(ADC1));
}

// 读取反电动势
int16_t Get_BEMF(void)
{
    ADC_RegularChannelConfig(ADC1, ADC_Channel_0, 1, ADC_SampleTime_55Cycles5);
    ADC_SoftwareStartConvCmd(ADC1, ENABLE);
    while(!ADC_GetFlagStatus(ADC1, ADC_FLAG_EOC));
    return (int16_t)(ADC_GetConversionValue(ADC1) - 2048);
}

void TIM2_Init_Config(void)
{
    RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM2, ENABLE);

    TIM_TimeBaseInitTypeDef TIM_TimeBaseStructure;
    NVIC_InitTypeDef NVIC_InitStructure;

    // 1ms 中断
    TIM_TimeBaseStructure.TIM_Period        = 72-1;
    TIM_TimeBaseStructure.TIM_Prescaler     = 1000-1;
    TIM_TimeBaseStructure.TIM_ClockDivision = TIM_CKD_DIV1;
    TIM_TimeBaseStructure.TIM_CounterMode   = TIM_CounterMode_Up;
    TIM_TimeBaseInit(TIM2, &TIM_TimeBaseStructure);

    NVIC_InitStructure.NVIC_IRQChannel                   = TIM2_IRQn;
    NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 1;
    NVIC_InitStructure.NVIC_IRQChannelSubPriority        = 1;
    NVIC_InitStructure.NVIC_IRQChannelCmd                = ENABLE;
    NVIC_Init(&NVIC_InitStructure);

    TIM_ITConfig(TIM2, TIM_IT_Update, ENABLE);
    TIM_Cmd(TIM2, ENABLE);
}

// 中断服务函数
void TIM2_IRQHandler(void)
{
    if(TIM_GetITStatus(TIM2, TIM_IT_Update) != RESET)
    {
        TIM_ClearITPendingBit(TIM2, TIM_IT_Update);

        time_count++;         // 系统计时
        bemf = Get_BEMF();    // 采样反电动势
        zero_point_control();// 无感过零换相
    }
}

void zero_point_control(void)
{
    if(zero_state == 0)
    {
        // 过零检测
        if( (step_reg%2==0 && bef>0 && bemf<0) || (step_reg%2==1 && bef<0 && bemf>0) )
        {
            zero_count ++;
            if(zero_count >= 4)  // 滤波4次确认过零
            {
                zero_count = 0;
                zero_state = 1;
                step_T = time_count;
                time_count = 0;
            }
        }
        else
        {
            zero_count = 0; // 没检测到就清零
        }
    }
    else if(zero_state == 1)
    {
        if(time_count > step_T/2)
        {
            zero_state = 0;
            step_reg = (step_reg + 1) % 6; // 换相
        }
    }

    bef = bemf; // 更新上次值
}

        case 1: B_H_SET; C_L_SET; break;
        case 2: C_H_SET; A_L_SET; break;
        case 3: A_H_SET; C_L_SET; break;
        case 4: B_H_SET; A_L_SET; break;
        case 5: C_H_SET; B_L_SET; break;
    }
}

#define B_L_CLR    GPIO_ResetBits(GPIOB, GPIO_Pin_14)

#define ALL_OFF()  do{A_H_CLR;B_H_CLR;C_H_CLR;A_L_CLR;B_L_CLR;C_L_CLR;}while(0)

//==================== 全局变量 ====================
uint8_t  zero_state = 0;
uint8_t  step_reg   = 0;
uint32_t time_count = 0;
uint32_t step_T     = 0;
uint8_t  zero_count = 0;

int16_t  bef        = 0;
int16_t  bemf       = 0;

uint32_t time_set   = 5;  // 启动延时

//==================== 延时 ====================
void Delay_ms(uint32_t ms) {
  uint32_t i,j;
  for(i=ms; i>0; i--)
    for(j=7200; j>0; j--);
}

//==================== ADC 初始化（反电动势采样 PA0）====================
void ADC_Init_Config(void)
{
  RCC_APB2PeriphClockCmd(RCC_APB2Periph_ADC1 | RCC_APB2Periph_GPIOA, ENABLE);

  GPIO_InitTypeDef GPIO_InitStructure;
  ADC_InitTypeDef ADC_InitStructure;

  GPIO_InitStructure.GPIO_Pin = GPIO_Pin_0;
  GPIO_InitStructure.GPIO_Mode = GPIO_Mode_AIN;
  GPIO_Init(GPIOA, &GPIO_InitStructure);

  ADC_InitStructure.ADC_Mode = ADC_Mode_Independent;
  ADC_InitStructure.ADC_ScanConvMode = DISABLE;
  ADC_InitStructure.ADC_ContinuousConvMode = DISABLE;
  ADC_InitStructure.ADC_ExternalTrigConv = ADC_ExternalTrigConv_None;
  ADC_InitStructure.ADC_DataAlign = ADC_DataAlign_Right;
  ADC_InitStructure.ADC_NbrOfChannel = 1;
  ADC_Init(ADC1, &ADC_InitStructure);

  ADC_Cmd(ADC1, ENABLE);
  ADC_ResetCalibration(ADC1);
  while(ADC_GetResetCalibrationStatus(ADC1));
  ADC_StartCalibration(ADC1);
  while(ADC_GetCalibrationStatus(ADC1));
}

//==================== 读取反电动势 ====================
int16_t Get_BEMF(void)
{
  ADC_RegularChannelConfig(ADC1, ADC_Channel_0, 1, ADC_SampleTime_55Cycles5);
  ADC_SoftwareStartConvCmd(ADC1, ENABLE);
  while(!ADC_GetFlagStatus(ADC1, ADC_FLAG_EOC));
  return (int16_t)(ADC_GetConversionValue(ADC1) - 2048);
}

//==================== 定时器2中断（1ms）====================
void TIM2_Init_Config(void)
{
  RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM2, ENABLE);

  TIM_TimeBaseInitTypeDef TIM_TimeBaseStructure;
  NVIC_InitTypeDef NVIC_InitStructure;

  TIM_TimeBaseStructure.TIM_Period = 72-1;
  TIM_TimeBaseStructure.TIM_Prescaler = 1000-1;
  TIM_TimeBaseStructure.TIM_ClockDivision = TIM_CKD_DIV1;
  TIM_TimeBaseStructure.TIM_CounterMode = TIM_CounterMode_Up;
  TIM_TimeBaseInit(TIM2, &TIM_TimeBaseStructure);

  NVIC_InitStructure.NVIC_IRQChannel = TIM2_IRQn;
  NVIC_InitStructure.NVIC_IRQChannelPreemptionPriority = 1;
  NVIC_InitStructure.NVIC_IRQChannelSubPriority = 1;
  NVIC_InitStructure.NVIC_IRQChannelCmd = ENABLE;
  NVIC_Init(&NVIC_InitStructure);

  TIM_ITConfig(TIM2, TIM_IT_Update, ENABLE);
  TIM_Cmd(TIM2, ENABLE);
}

//==================== 定时器中断服务函数 ====================
void TIM2_IRQHandler(void)
{
  if(TIM_GetITStatus(TIM2, TIM_IT_Update) != RESET)
  {
    TIM_ClearITPendingBit(TIM2, TIM_IT_Update);
    time_count++;
    bemf = Get_BEMF();    // ADC采样
    zero_point_control(); // 无感控制
  }
}

//==================== 换相输出====================
void Motor_Output(void)
{
  ALL_OFF();
  switch(step_reg)
  {
    case 0: C_H_SET; B_L_SET; break;
    case 1: A_H_SET; C_L_SET; break;
    case 2: B_H_SET; A_L_SET; break;
    case 3: C_H_SET; B_L_SET; break;
    case 4: A_H_SET; C_L_SET; break;
    case 5: B_H_SET; C_L_SET; break;
  }
}

//==================== zero_point_control====================
void zero_point_control(void)
{
  if(zero_state == 0)
  {
    zero_count ++;
    if(zero_count >= 4)
    {
      zero_count = 0;
      zero_state = 1;
      step_T = time_count;
      time_count = 0;  // 补了分号
    }
  }
  else
  {
    zero_count = 0; 
  }

  if(zero_state == 1)
  {
    if(time_count > step_T/2)
    {
      zero_state = 0;
      step_reg = (step_reg+1)%6;
    }
  }
}

//==================== MAIN 函数 ====================
int main(void)
{
  // 1. 时钟
  RCC_APB2PeriphClockCmd(RCC_APB2Periph_TIM1, ENABLE);
  RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA|RCC_APB2Periph_GPIOB, ENABLE);

  // 2. GPIO 初始化
  GPIO_InitTypeDef GPIO_InitStructure;
  GPIO_InitStructure.GPIO_Mode = GPIO_Mode_Out_PP;
  GPIO_InitStructure.GPIO_Pin = GPIO_Pin_8|GPIO_Pin_9|GPIO_Pin_10;
  GPIO_InitStructure.GPIO_Speed = GPIO_Speed_50MHz;
  GPIO_Init(GPIOA, &GPIO_InitStructure);

  GPIO_InitStructure.GPIO_Pin = GPIO_Pin_13|GPIO_Pin_14|GPIO_Pin_15;
  GPIO_Init(GPIOB, &GPIO_InitStructure);

  ALL_OFF();

  // ADC + TIM2 
  ADC_Init_Config();
  TIM2_Init_Config();


  while(1)
  {
    Delay_ms(time_set);
    step_reg++;
    step_reg %= 6;
    Motor_Output();  // 换相输出
  }
}

 磁场旋转⻆度：每次旋转120度，通过程序步序加⼆实现。
 实验现象观察：换向后峰值增⼤，转⼦加速形成翻倍效果。

 余弦曲线⾏为：绝对值逐渐减⼩，过零点后随⻆度增加⽽增加。
 电压变化规律：在0度达到最⼤值，过程呈现周期性重复。

 强制换向启动：⾸先强制换向到第零步并持续100毫秒，确保电机转到位且振荡消失。
 两步换向策略：直接换向到第⼆步，线圈领先转⼦120度，保持5毫秒使转⼦转过约30度。
